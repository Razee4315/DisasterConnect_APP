# Contributing

Thank you for your interest in contributing to DisasterConnect. This document outlines the process for contributing to this project.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch from `main` for your work
4. Make your changes
5. Test your changes locally
6. Push to your fork and submit a pull request

## Development Setup

```bash
git clone https://github.com/<your-username>/DisasterConnect_APP.git
cd DisasterConnect_APP
bun install
bun run tauri dev
```

### Requirements

- Bun 1.0+
- Rust 1.70+
- Tauri v2 system dependencies

## Pull Request Guidelines

- Keep pull requests focused on a single change
- Write clear commit messages that explain the "why", not just the "what"
- Make sure the project builds without errors before submitting
- Update documentation if your changes affect usage or setup
- Follow the existing code style and conventions in the project

## Reporting Bugs

Open an issue on GitHub with the following information:

- A clear description of the bug
- Steps to reproduce
- Expected behavior vs actual behavior
- Your OS and app version
- Screenshots if applicable

## Suggesting Features

Open an issue with the "feature request" label. Include:

- A clear description of the feature
- The problem it solves or the use case it addresses
- Any relevant context or examples
